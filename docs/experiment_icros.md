# ICROS 학술대회 실험 결과 분석 (MPPI Controller)

## 1. 개요 (Introduction)
본 문서는 MPPI(Model Predictive Path Integral) 로컬 플래너를 적용한 로봇의 주행 성능 및 연산 부하(CPU Usage)를 평가하기 위한 실험 결과를 정리합니다. 총 5회의 주행 시나리오(Trial 01~05)에 대하여 주행 소요 시간(Time to Goal)과 CPU 점유율 데이터를 수집하였으며, 이를 바탕으로 MPPI 제어기의 실효성 및 시스템 요구사항을 분석합니다.

## 2. 실험 결과 (Experimental Results)

실험에서 수집된 rosbag 데이터(주행 시간)와 cpu_log 데이터를 취합한 결과는 다음 표와 같습니다. 

| Trial | Time to Goal (s) | Avg CPU (%) | Max CPU (%) | 비고 |
|:-----:|:----------------:|:-----------:|:-----------:|:----|
| 01    | 33.56            | 6.00        | 6.00        | CPU 로깅 데이터 부족/초기화 이슈 |
| 02    | 39.07            | 23.13       | 35.00       | - |
| 03    | 34.37            | 25.68       | 42.40       | 최대 CPU 부하 발생 |
| 04    | 32.60            | N/A         | N/A         | CPU 로깅 실패 |
| 05    | 28.98            | 25.41       | 36.90       | 최단 시간 도달 |

> [!NOTE]
> Trial 01은 CPU 점유율이 비정상적으로 낮게 측정되었고, Trial 04는 CPU 로그가 기록되지 않았습니다. 분석 시에는 정상적으로 기록된 Trial 02, 03, 05 데이터를 위주로 참고하는 것이 타당합니다.

## 3. 데이터 분석 (Data Analysis)

### 3.1 주행 성능 (Time to Goal) 분석
- **평균 주행 소요 시간**: 약 33.7초 (5회 평균)
- **최단 및 최장 시간**: 최단 28.98초 (Trial 05), 최장 39.07초 (Trial 02)
- 주행 시간에 약 10초 내외의 편차가 존재합니다. 이는 MPPI의 샘플링 기반 경로 탐색 특성이나 동적 장애물 여부, 혹은 초기 위치(Pose) 설정 오차에 기인한 것으로 보입니다.

### 3.2 연산 부하 (CPU Usage) 분석
- **평균 CPU 점유율**: 정상 기록된 Trial(02, 03, 05) 기준 약 **24.7%**
- **최대 CPU 점유율 (Peak)**: **42.4%** (Trial 03)
- MPPI 알고리즘은 수많은 경로(Trajectory) 롤아웃을 동시에 병렬 연산하므로 연산 부하가 큰 편입니다. 평균 25%의 CPU를 지속적으로 점유하고 피크 시 40% 이상 치솟는 것은 시스템 자원에 적지 않은 부담을 줍니다. 
- **시사점**: 저사양 임베디드 보드에서는 MPPI의 병목으로 인해 제어 주기가 지연될 리스크가 있습니다. 제어 주파수(Controller Frequency) 최적화, 샘플 수(Batch Size) 감소, 혹은 GPU 가속(CUDA 기반 MPPI) 도입 등 추가적인 튜닝이 요구될 수 있습니다.

## 4. 결론 (Conclusion)
MPPI 로컬 플래너는 목표 지점까지 성공적으로 로봇을 제어하지만, 시스템 CPU 리소스를 평균 25%가량 지속적으로 사용하며 최대 42% 이상의 부하를 발생시킵니다. 향후 실환경 적용 및 시스템 통합 시 롤아웃(Rollout) 샘플 수 최적화를 통해 연산량을 줄이거나 타 로컬 플래너(DWB 등)와의 정량적 비교 실험을 추가하여 논문을 보강할 것을 권장합니다.
