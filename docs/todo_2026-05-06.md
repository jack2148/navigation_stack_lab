# 내일 할 것 (2026-05-06)

## 핵심 문제 요약

`bringup_nav` 실행 시 Nav2 controller_server가 **SIGSEGV(Segmentation Fault)**로 죽음.
원인은 ARM64(Jetson) + ROS2 Humble 환경에서 `ros-humble-nav2-mppi-controller` 바이너리의 알려진 크래시 버그.

크래시 흐름:
```
MPPI critic 초기화 중 SIGSEGV
  → controller_server 죽음
  → map → odom TF 소멸 (AMCL도 영향)
  → global_costmap "Invalid frame ID map" 에러
  → Nav2 lifecycle 전체 종료
```

---

## 1순위: MPPI 바이너리 재설치

### 환경 확인
```bash
echo $ROS_DISTRO   # humble 이어야 함
uname -m           # aarch64 이어야 함
```

### 재설치
```bash
sudo apt update
sudo apt install --reinstall ros-humble-nav2-mppi-controller
```

재설치 후 10회 실행 테스트:
- SIGSEGV 없이 실행되면 성공
- 여전히 크래시 나면 소스 빌드 필요 (아래 참고)

### 재설치 후에도 크래시 시: 소스 빌드
```bash
mkdir -p ~/nav2_mppi_ws/src
cd ~/nav2_mppi_ws/src
git clone -b humble https://github.com/ros-navigation/navigation2.git
cd ~/nav2_mppi_ws
source /opt/ros/humble/setup.bash
rosdep install --from-paths src --ignore-src -r -y
colcon build --packages-up-to nav2_mppi_controller \
  --symlink-install \
  --cmake-args -DCMAKE_BUILD_TYPE=RelWithDebInfo
source install/setup.bash
```
이후 반드시 이 터미널에서 launch.
